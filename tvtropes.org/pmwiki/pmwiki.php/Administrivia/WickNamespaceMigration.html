Administrivia / Wick Namespace Migration

Go To

Edit Locked

  • Administrivia
  • Create New

In a perfect world, every work on TV Tropes would have been in its corresponding namespace right from the get-go. But in the words of Fast Eddie: "If we had had more forethought, that's the way we would have started five years ago."

Thus pages were moved from the Main/ namespace into their correct namespace, but the wicks still need work done. You can help by moving a work's article you happen to come across that is in the wrong place (see How to Move a Page for step-by-step instructions), or list it here. Careful, though. Sites all over the world link to our works' pages. We need to leave a redirect in place to service those links.

Advertisement:

Listed below are works that have been moved to the correct namespace, but still need their wicks changed to point to the new page. If you have any questions, you can refer to the Wick Migration thread. If a page is locked and not an archive, ask here.

See also Completed Namespace Migrations, Franchise Namespace Correction, and the Wick Hyphen Migration on the Hyphenated Titles page. There should be no more Main redirects that aren't on this list, cutlisted, or under discussion for transformation into disambigs, but that has only been confirmed up to "Blowing Stuff Up" and down from "Yuri". Here's the list of all Main redirects. Remember that not all redirects show up in the boxes in the Relateds listing. Some are just listed as wicks.

What to do when moving a work:
  1. Move content to the new namespace.note Check the namespace article to find out where to move it. (Main/WorkName → CorrectNamespace/WorkName)note Whenever possible, the work name should be a Wiki Word and need not use brackets unless it's a single word; please avoid hyphens outside of customized titles.
  2. Change any redirects to point to the new article location.note Add "?action=edit" at the end of the corresponding complete URL in the address bar to edit a redirect.
  3. If there is a discussion and/or reviews, make a request for them to be moved in the appropriate thread.
  4. Set the page type of the new page to "a work" (or "a creator or person" for the Creator/ namespace) — this is now automated.
  5. If the work title is non-standard and not already customized, go to Customize Wikiword to make a request.
  6. Correct wicks, starting with the indexes.note Do not correct an index wick before creating the new article, though; the index bar wouldn't display on the bottom of the page until the index is modified again.
  7. Doublecheck everything, then deposit in the Completed Namespace Migrations page and pat yourself on the back (be sure to keep additional wicks in check).
  8. Cut the Main namespaced version, if you're moving a redirect to its proper namespace.
  9. If, instead, you're changing wicks to ones with a different namespace and a different name, make a redirect to the page being moved to, in a version of the Main redirect's name in that new namespace, to catch inbounds, before cutting the Main redirect.
Advertisement:

It would also be helpful if you Namespace any other links to works that you notice.

Works in bold have 250 wicks or fewer to be moved.
